## 6. Layout Recommendations

### Grid system
- **Admin content max-width:** `max-w-7xl mx-auto` inside main — prevents ultra-wide tables on large monitors (Stripe pattern)  
- **Page spacing:** `space-y-8` between sections (up from `space-y-6`) for breathing room  
- **Card grid:** Dashboard KPIs `grid-cols-1 sm:2 lg:4 gap-4`; detail panels `lg:grid-cols-2 gap-6`

### Admin navigation
- **Desktop:** Keep 256px sidebar; add subtle `border-r` and active item left accent bar (Linear-style)  
- **Mobile:** Replace horizontal pill scroll with **sheet/drawer** nav (hamburger) — pills overflow and hide labels  
- **Breadcrumb optional:** For deep admin context (e.g. Admin → Candidates → Edit)

### Member navigation
- Single-row header on mobile: logo | nav tabs | actions  
- Remove full-page gradient; use flat `bg-background` with subtle top border — more “official portal”  
- Ballot content: `max-w-3xl` centered for focus (Notion-like reading width)

### White space
- Increase table cell padding (`py-3 px-4`)  
- Dialog forms: consistent `space-y-4` with labeled groups  
- Empty states: maintain `p-12` but add optional primary CTA slot

## 8. Dashboard Redesign Plan

**Reference:** Vercel Analytics + Stripe Dashboard — calm, data-first, no clutter.

### Structure
```
PageHeader: "Dashboard" + election status badge + last updated timestamp
Row 1: 4 KPI StatCards (Members, Candidates, Positions, Votes Cast)
Row 2: Primary chart — turnout over time OR position comparison bar chart (full width)
Row 3: Two columns
  Left: Participation breakdown (donut or stacked bar — Completed / Partial / None)
  Right: Position turnout list with inline progress bars
Row 4 (optional): Recent activity feed (last 5 audit events) — reinforces trust
```

### UX improvements
- **Live indicator:** Pulsing dot when election ACTIVE + “Updated Xs ago”  
- **Error state:** Full `EmptyState` with retry button, not red text  
- **No election:** Guided empty — “Create election” CTA linking to Elections page  
- **Motion:** Subtle stagger on KPI cards only (150ms total); no bounce  
- **Numbers:** Tabular nums, formatted with locale  

### Visual
- KPI cards: white/card bg, subtle border, no heavy shadow  
- Remove emerald/amber icon colors → semantic success/warning tokens  
- Chart: single primary color bars; secondary series muted

## 11. Accessibility Improvements

| Area | Action |
|------|--------|
| **Color contrast** | Audit all `muted-foreground` on `muted` bg; fix success/warning badge pairs |
| **Keyboard** | All ballot candidates focusable; Enter/Space selects; confirm dialog traps focus |
| **Focus visible** | Ensure `ring-2 ring-ring ring-offset-2` on all interactive elements (shadcn default — verify not overridden) |
| **Icon buttons** | `aria-label` on every edit/delete/upload action |
| **Ballot selection** | `role="radiogroup"` per position; `aria-checked` / `aria-pressed` on candidates |
| **Headings** | One `h1` per page; position titles as `h2`; no skipped levels |
| **Live regions** | `aria-live="polite"` on toast container; vote success announced |
| **Forms** | Associate labels via `htmlFor`; expose errors with `aria-describedby` |
| **Charts** | Provide table fallback or `aria-label` summary for screen readers |
| **Motion** | Respect `prefers-reduced-motion` — disable Framer Motion transforms |

**Target:** WCAG 2.1 AA for core flows (login, ballot, admin dashboard).
